โปรแกรม IMPS Service ถูกออกแบบขึ้นมาเพื่อทำหน้าที่เป็น Middleware จัดการด่านชั่งน้ำหนักยานพาหนะ โดยเชื่อมสัญญาณแบบ Real-time และลดความซับซ้อนของการเชื่อมต่อฮาร์ดแวร์ประเภทต่างๆ ให้ทำงานร่วมกันอย่างอัตโนมัติ

🔌 Hardware Abstraction
รองรับการดึงข้อมูลจากบอร์ดชั่งน้ำหนักสองยี่ห้อหลัก (DataLogger และ InterComp) ผ่านทาง WebSocket Client พร้อมระบบพยายามเชื่อมต่อใหม่แบบ Exponential Backoff
👀 Image Processing & OCR
เมื่อรถชั่งน้ำหนัก ระบบจะค้นหารูปถ่ายจากกล้องที่จับคู่เวลาและเลน สั่งรันตรวจจับ OCR อ่านค่าป้ายทะเบียน จากนั้นตัดแบ่งเฉพาะส่วนป้าย (Crop) ด้วย Sharp และอัปโหลดไปยังคลาวด์
📏 3D & Wheel Integration
ดึงรายละเอียดขนาดกว้าง ยาว สูง และสถานะสูงเกินกำหนดจากเครื่องสแกน 3D พร้อมติดต่อกับบอร์ด Pico เพื่อตรวจประเภทของล้อ/ยางในเลน 1
🔔 Real-time Outputs
อัปเดตจอแสดงผล LED (VMS) หน้าด่านชั่งทันทีที่ทราบน้ำหนัก และทำการส่งสัญญาณ WebSocket แจ้งเตือนแผงควบคุม Dashboard พร้อมส่งข้อมูล API ส่วนกลาง
💡
จุดเด่นด้านเสถียรภาพการทำงาน: ระบบมีการทำ Database Configuration Polling ทุกๆ 5 วินาที เพื่อดูว่าตัวแปรตั้งค่าเปลี่ยนหรือไม่ หากเปลี่ยนระบบจะ Auto-restart อุปกรณ์ควบคุมหน้างานทันทีแบบ On-the-fly โดยไม่ต้องปิดเซิร์ฟเวอร์หลัก

🧭 ขั้นตอนการประมวลผลคันรถชั่ง

เส้นทางการไหลของข้อมูลเมื่อยานพาหนะวิ่งผ่านจุดรับสัญญาณชั่งน้ำหนัก (Weighing Event Pipeline)

ขั้นตอนที่ 1: ตรวจจับสัญญาณการเข้าด่านชั่ง (Trigger)
เมื่อรถเริ่มเหยียบชั่ง สัญญาณ Trigger Event (Start) จะส่งผ่าน WebSocket เข้ามา ระบบสั่งบันทึกรูปภาพจากกล้องทันที โดยสร้างชื่อภาพแบบระบุประเภท เลน และเวลาแสตมป์ละเอียด เพื่อเตรียมพร้อมไว้
ขั้นตอนที่ 2: จำแนกและบันทึกประวัติการชั่งเบื้องต้น (Weighing Event)
เมื่อรถวิ่งผ่านพ้น บอร์ดชั่งส่งข้อมูลรวมเข้ามา ระบบจะจำแนกประเภทรถ (Vehicle Classification) ตามจำนวนเพลาและระยะฐานล้อ, คำนวณค่าสะสมน้ำหนักบรรทุกล้อเดี่ยวมาตรฐาน (ESAL), และบันทึกประวัติลงฐานข้อมูลทันที
ขั้นตอนที่ 3: ตรวจสอบและดึงข้อมูลบูรณาการ (Metadata Enhancement)
ระบบทำการเชื่อมโยงข้อมูลในเลนที่เกี่ยวข้อง โดยไปขอข้อมูลยางเดี่ยว/ยางคู่จาก Raspberry Pi Pico และสอบถามขนาดมิติตัวรถจาก 3D Scanner API โดยใช้เลขป้ายทะเบียนและเวลามาจับคู่บันทึก
ขั้นตอนที่ 4: งานเบื้องหลังอัปเดตรูปป้ายทะเบียน (Asynchronous LPR & OCR)
สคริปต์ Background Worker ค้นหาไฟล์ภาพชั่วคราวในโฟลเดอร์ โดยวนลูปหาด้วย Smart Retry 3 ครั้ง หากได้ภาพแล้วจะส่งไปดึงป้ายทะเบียนด้วย OCR API, ตัดรูปป้ายทะเบียนเก็บในระบบ และอัปเดตชื่อทะเบียนพร้อมพาธรูปภาพลง DB
ขั้นตอนที่ 5: ส่งข้อมูลผลลัพธ์ออกระบบ (Transmission)
ส่งผลลัพธ์ผ่านจอ LED (VMS) หน้าด่านชั่งทันที, ส่งกระจายเรียลไทม์ผ่าน WebSocket ไปที่ Dashboard คอนโซล และส่งบันทึกสุดท้ายขึ้น API ส่วนกลางในคลาวด์

🧠 ลอจิกอัจฉริยะและการวิเคราะห์ข้อมูล (Core Logic)

คำอธิบายกลไกตรรกะในส่วนการประมวลผลคำนวณและตัวคัดกรองความผิดพลาด

🚗 รถวิ่งคร่อมเลน (Straddling Merge Logic)

เมื่อรถขนาดใหญ่วิ่งคร่อมข้ามระว่างสองเลนชั่ง บอร์ดชั่งจะจับได้เป็นยานพาหนะแยกเลน 2 เลนที่มีจำนวนเพลาเท่ากันแต่มีน้ำหนักข้างเดียว (ซ้ายหรือขวา) ในเวลาห่างกันไม่เกิน 3 วินาที (ตั้งค่าได้ใน straddling_time_diff)

ระบบจะเก็บข้อมูลคันแรกไว้ใน Buffer ชั่วคราว เมื่อคันที่สองตามมาจะทำการ **ควบรวมข้อมูล (Merge)** เข้าด้วยกัน เอาน้ำหนักล้อซ้ายจากเลนซ้ายมารวมกับล้อขวาของเลนขวา ปรับปรุงค่า GVW และสถิติให้กลายเป็นรถคันเดียวอย่างเที่ยงตรง

🚌 การคัดกรองรถส่วนบุคคลและรถโดยสาร (Bus/Passenger Exclusion Filter)

เพื่อลดขยะข้อมูลในคลาวด์ ด่านชั่งจะเก็บเฉพาะรถบรรทุกเชิงพาณิชย์เท่านั้น ระบบมีระบบคัดกรอง 2 ชั้น:

  • ชั้นแรก (Wheelbase): ตรวจสอบฐานล้อเพลาคู่แรก ถ้าต่ำกว่ากำหนดและเข้าข่ายเป็นรถโดยสาร (Bus) ระบบจะไม่สั่งดำเนินการต่อ
  • ชั้นสอง (License Plate): หลังอ่านตัวอักษร OCR ถ้าพบตัวอักษรทะเบียนรถเก๋ง (กข, หป) หรือหมวดขึ้นต้นด้วยตัวเลขที่เป็นรถบัส (10-49 ตามมาตรฐานกรมการขนส่งทางบก) ระบบจะลบประวัติของรถคันนั้นๆ ในฐานข้อมูลทิ้งทันที

📐 การวิเคราะห์ความสูง 3D (3D Dimensions Validation)

การเชื่อมต่อ API ของ 3D Scanner เพื่อบันทึกขนาดจะดึงประวัติตัวรถย้อนหลัง-ล่วงหน้าในช่วง `±5 นาที` และนำทะเบียนมา Normalize ตัดอักษรพิเศษออกทั้งหมดเพื่อให้เปรียบเทียบได้ถูกต้อง จากนั้นนำค่าความสูงที่สแกนได้เทียบกับ THREE_DIMENSION_MAXIMUM_HEIGHT หากสูงเกินจะเซ็ตเตือนสถานะสูงเกินใน Warning Map ของตาราง 3D

💾 โครงสร้างฐานข้อมูลและการตั้งค่า (DB & Configuration)

รายชื่อตารางหลักในระบบฐานข้อมูล MySQL และตัวแปรตั้งค่าระบบของไฟล์ `.env` ที่ใช้งานปัจจุบัน

🔑 ข้อมูลตารางฐานข้อมูลหลัก (Core Tables)

ชื่อตาราง (Table Name) รายละเอียดและฟังก์ชันการเก็บบันทึกข้อมูล
configuration เก็บบันทึกข้อมูลการตั้งค่ากายภาพด่านชั่ง, กล้อง, ป้ายไฟ, ข้อกำหนด และจำนวนวันเก็บล็อกภาพ
vehicles ประวัติน้ำหนักรวม (GVW), ความเร็ว, เลนวิ่ง, แสตมป์เวลา และสถานะการบรรทุกน้ำหนักเกินกฎ DLT
axles น้ำหนักรายเพลา, น้ำหนักข้างซ้าย-ขวา, ระยะฐานล้อ (wheelbase), และข้อมูลประเภทยางเดี่ยว/ยางคู่
plates ตัวอักษรทะเบียน, จังหวัด, พาธรูปภาพป้ายทะเบียนแบบ LPR และรูปถ่ายป้ายพรีวิวที่ผ่านการ Crop
three_dimension เก็บบันทึกขนาด กว้าง ยาว สูง และพาธรูปภาพสแกน 3D ของยานพาหนะ
snapshots ตารางพักเก็บข้อมูลรูปภาพที่กล้องดึงมา รอการรันวิเคราะห์ในเบื้องหลัง (Background Worker)
