# แผนการพัฒนาและจุดที่ควรปรับปรุงในอนาคต (System Roadmap)

จากการตรวจสอบโครงสร้างระบบ IMPS Service ปัจจุบัน นี่คือจุดที่ควรพิจารณาปรับปรุงในระยะยาวเพื่อความเสถียรและความง่ายในการดูแลรักษาครับ:

## 1. ระบบจัดการคิวรูปภาพ (Memory Scalability)
*   **ปัญหาปัจจุบัน:** `snapshotRegistry` เก็บข้อมูลรูปล่าสุดไว้ในตัวแปรในหน่วยความจำ (RAM) ของ Node.js โดยตรง
*   **ความเสี่ยง:** หากระบบ Restart ข้อมูลในคิวจะหายทันที และหากรันหลาย Instance (Cluster Mode) แต่ละตัวจะมองไม่เห็นข้อมูลของกันและกัน
*   **แนวทางแก้ไข:** เปลี่ยนไปใช้ **Redis** เป็นที่เก็บ Snapshot Metadata ชั่วคราวแทน เพื่อให้รองรับการขยายตัว (Scalability) และข้อมูลไม่หายเวลา Restart

## 2. การจัดการภาพถ่ายแบบ Full Asynchronous
*   **ปัญหาปัจจุบัน:** การดึงรูปภาพจากกล้อง (`axios.get`) ทำงานอยู่ใน Main Process แม้จะใช้ `Promise.all` แต่ถ้ากล้องตอบสนองช้ามาก อาจส่งผลต่อประสิทธิภาพโดยรวม
*   **แนวทางแก้ไข:** ใช้ระบบ **Message Queue (เช่น RabbitMQ หรือ BullMQ)** ในการสั่งจับภาพ เพื่อแยกส่วนการติดต่อกล้อง (Network I/O) ออกจากส่วนประมวลผลข้อมูลรถ (Data Processing) อย่างเด็ดขาด

## 3. ความปลอดภัยและการจัดการฐานข้อมูล (Database Integrity)
*   **ปัญหาปัจจุบัน:** มีการเขียน SQL Query แบบดิบ (Raw SQL) กระจายอยู่หลายไฟล์ ซึ่งยากต่อการดูแลรักษาหากมีการเปลี่ยน Schema
*   - **แนวทางแก้ไข:** เปลี่ยนไปใช้ **ORM (เช่น Sequelize หรือ Prisma)** เพื่อช่วยจัดการเรื่องความสัมพันธ์ของข้อมูล (Relationships) และช่วยให้การทำ Migration ฐานข้อมูลเป็นระบบมากขึ้น

## 4. ระบบบันทึกเหตุการณ์ (Structured Logging)
*   **ปัญหาปัจจุบัน:** ใช้ `console.log` และ `console.error` เป็นหลัก ซึ่งยากต่อการค้นหาต้นตอปัญหาเมื่อระบบทำงานไปนานๆ
*   **แนวทางแก้ไข:** เปลี่ยนไปใช้ Library สำหรับ Log โดยเฉพาะ เช่น **Winston** หรือ **Pino** เพื่อบันทึก Log ลงไฟล์แยกตามวัน และรองรับการส่ง Log ไปยังระบบ Monitor ส่วนกลาง (เช่น ELK Stack)

## 5. การจัดการป้ายทะเบียนภาษาไทย (OCR Enhancement)
*   **ปัญหาปัจจุบัน:** Logic การกรองป้ายทะเบียน (ที่ผมพบ) ยังมีความเสี่ยงที่จะกรองรถป้ายไทยปกติออกหาก OCR อ่านได้แม่นเกินไป
*   **แนวทางแก้ไข:** พัฒนา Regex หรือ Logic ที่รองรับรูปแบบป้ายทะเบียนไทยทุกหมวด (เช่น รถส่วนบุคคล, รถสาธารณะ, ป้ายกราฟิก) เพื่อลดการ Early Return หรือการข้ามการบันทึกข้อมูลโดยไม่จำเป็น

## 6. ระบบตรวจสอบสถานะอุปกรณ์ (Health Check)
*   **ปัญหาปัจจุบัน:** ระบบจะรู้ว่ากล้องหรือ WIM มีปัญหาต่อเมื่อเกิด Error ขึ้นขณะทำงานเท่านั้น
*   **แนวทางแก้ไข:** เพิ่มฟังก์ชัน **Health Check/Heartbeat** เพื่อตรวจสอบสถานะการเชื่อมต่อกับอุปกรณ์ทุกอย่างเป็นระยะ และแจ้งเตือนผ่าน Line หรือ Email เมื่อพบอุปกรณ์ Offline
