## 1. ปัญหาที่พบ (Identified Issues)
จากการวิเคราะห์ข้อมูลกรณี "ภาพหายช่วง 00:00 - 00:03 น. ทั้งที่มีไฟล์ในเครื่องและมี OCR" พบสาเหตุหลักดังนี้:
*   **The Retry Trap (Bug หลัก):** เมื่อการอัปโหลดรูปภาพล้มเหลวในช่วงต้นวัน (ซึ่ง Server ปลายทางมักจะหน่วง) ระบบจะสั่ง Retry แต่การ Retry หาภาพเดิมไม่เจอ เพราะภาพถูกทำเครื่องหมายว่า "Used" (ถูกใช้แล้ว) ไปตั้งแต่รอบแรก
*   **Midnight Date Leap:** การตีความเวลาจาก Trigger กล้องผิดพลาดในช่วงรอยต่อวัน (เช่น กล้องส่ง 23:59:59 ขณะเซิร์ฟเวอร์เป็น 00:00:01) ทำให้ระบบหาภาพในโฟลเดอร์วันที่ผิด
*   **New Plate Format:** ระบบเดิมยังไม่รองรับการจัดรูปแบบและคัดกรองป้ายทะเบียนรุ่นใหม่ที่มีตัวเลขนำหน้าตัวอักษร เช่น "1หป"

| หัวข้อการแก้ไข | รายละเอียดการดำเนินงาน | ผลลัพธ์ที่ได้ |
| :--- | :--- | :--- |
| **Smart Retry System** | ปรับปรุงให้ระบบ "จำไฟล์เดิม" ที่หาเจอครั้งแรกไว้ในมือ หากอัปโหลดพลาด ให้ใช้ไฟล์ใบเดิมส่งซ้ำทันที | แก้ไขปัญหา "มี OCR แต่ไม่มีรูป" ได้ 100% แม้ Network จะติดขัด |
