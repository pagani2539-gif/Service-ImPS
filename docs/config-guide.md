# คู่มือตั้งค่า: ตาราง `configuration` (คร่อมเลน + รถไหลทาง)

ค่าตั้งทั้งหมดอยู่ใน **1 แถว** ของตาราง `configuration` — ระบบอ่านใหม่ทุก 5 วินาที
→ **แก้แล้วมีผลทันที ไม่ต้อง restart**

---

## ค่าที่ตั้งได้

| คอลัมน์ | ชนิด | default | หมายเหตุ |
|---|---|---|---|
| `straddling_axle_tol` | INT | 3 | ยอมจำนวนเพลาต่างกันได้กี่เพลา |
| `straddling_time_diff` | INT | 3 | วินาที — หน้าต่างเวลาจับคู่ |
| `straddling_speed_diff` | INT | 15 | กม./ชม. — ความเร็วต่างได้ |
| `straddling_wheelbase_diff` | INT | 30 | ซม. — ฐานล้อต่างได้ |
| `straddling_zero_kg` | INT | 100 | กก. — เกณฑ์ตัดสิน "ฝั่งศูนย์" ต่อล้อ |
| `mirror_edge_zones` | TEXT (JSON) | ว่าง = ปิด | โซนขอบถนน สำหรับ mirror รถไหลทาง |

> 5 ตัวแรกมีค่า default อัตโนมัติ ทำงานได้เลย — มีแค่ **`mirror_edge_zones`** ที่ต้องตั้งเองครั้งเดียว

---

## 1. รูปแบบ `mirror_edge_zones`

JSON array ของ `{lane, side}`:
- `lane` = หมายเลขเลน (ตามฟิลด์ `LaneNo` ที่ WIM ส่งมา)
- `side` = ขอบถนนอยู่ด้านไหนของเลนนั้น — `"L"` (ซ้าย) หรือ `"R"` (ขวา)

ตัวอย่าง: `[{"lane":1,"side":"L"},{"lane":3,"side":"R"}]`
= "เลน 1 ขอบ(เกาะกลาง)อยู่ซ้าย, เลน 3 ขอบ(ไหล่ทาง)อยู่ขวา"

> ตั้ง **เฉพาะเลนริมที่ติดขอบจริง** — เลนกลางไม่ต้องใส่
> (รถที่ไหลออกเลนกลางจะไปลงเลนข้างๆ = เป็น straddle merge ไม่ใช่ mirror)

---

## 2. หาค่าที่ถูกต้อง (เลือก 1 วิธี)

### วิธี A — จากผังหน้างาน

**กฎผังไทยมาตรฐาน (ขับชิดซ้าย, เลน 1 = เลนช้า = ซ้ายสุด, เลขเพิ่มไปทางขวา):**
```
(ซ้าย) ไหล่ทาง/ขอบถนน │ เลน 1(ช้า) │ เลน 2 │ ... │ เลนสูงสุด(เร็ว) │ เกาะกลาง (ขวา)
          ◀ L                                                       R ▶
```
- **เลน 1 (ช้า/ซ้ายสุด) → `"L"`** เสมอ
- **เลนเลขสูงสุด (เร็ว/ขวาสุด) → `"R"`** เสมอ
- เลนกลางไม่ต้องตั้ง

> หลักคิด: `side` คือ "ล้อไหลออกไปทาง**ซ้ายหรือขวา**ของเลนนั้น" — ไม่เกี่ยวกับชื่อ (ขอบถนน/เกาะกลาง เป็นขอบเหมือนกัน)
> ถ้าหมายเลขเลนเรียงกลับทาง (ขวาสุด = เลน 1) ก็สลับเป็น เลน 1 = `"R"`, เลนสูงสุด = `"L"`

### วิธี B — ดูจาก log (แม่นยำ ไม่ต้องเดา) ← แนะนำ
ดูบรรทัด `[EdgeMirror][Skip]` หรือ `[Straddling][Orphan]` ของรถที่ไหลออกขอบ ที่ช่อง `ZeroSig`:

| ZeroSig | แปลว่า | ตั้ง `side` |
|---|---|---|
| `A4:R0 A5:R0` (ขวามีค่า, ซ้าย=0) | ล้อ**ซ้าย**หาย | `"L"` |
| `A4:L0 A5:L0` (ซ้ายมีค่า, ขวา=0) | ล้อ**ขวา**หาย | `"R"` |

จดว่าเลนไหนมี orphan แบบนี้ + ฝั่งไหนหาย → ได้ `{lane, side}` ครบ

---

## 3. ต่อฐานข้อมูล

ข้อมูลต่ออยู่ในไฟล์ `.env` ของ runtime (ตัวอย่าง: host `localhost`, user `imps`, db `imps_db` — ดูรหัสจริงใน `.env`)

**MySQL CLI:**
```bash
mysql -u imps -p imps_db
```

**หรือโปรแกรม GUI:** HeidiSQL / DBeaver / MySQL Workbench → ต่อด้วย host/user/pass/db เดียวกัน → เปิดตาราง `configuration`

---

## 4. คำสั่งตั้งค่า (ตาราง `configuration` มีแถวเดียว)

**3 เลน (เลน 1 ซ้าย, เลน 3 ขวา):**
```sql
UPDATE configuration
SET mirror_edge_zones = '[{"lane":1,"side":"L"},{"lane":3,"side":"R"}]';
```

**5 เลน (เลน 1 ซ้าย, เลน 5 ขวา):**
```sql
UPDATE configuration
SET mirror_edge_zones = '[{"lane":1,"side":"L"},{"lane":5,"side":"R"}]';
```

**เลนเดียว (เลน 1 ซ้าย):**
```sql
UPDATE configuration
SET mirror_edge_zones = '[{"lane":1,"side":"L"}]';
```

> ⚠️ JSON ครอบด้วย single quote `'...'` · ใช้ double quote `"` ข้างใน · `side` ตัวใหญ่ `L`/`R`

---

## 5. ตรวจว่าค่าเข้าแล้ว

**อ่านกลับ:**
```sql
SELECT mirror_edge_zones FROM configuration;
```

**ดู log (ไม่ต้อง restart):** พอมีรถไหลออกขอบ จะเปลี่ยนจาก
`[EdgeMirror][Skip] reason=เลนนี้ไม่ได้ตั้งขอบ...` → `[EdgeMirror] ... mirror เพลา [4,5] ...`

---

## 6. ปรับ / ปิด

**ปิดฟีเจอร์ (กลับ default):**
```sql
UPDATE configuration SET mirror_edge_zones = '[]';
```

**แก้เลน/ฝั่ง:** UPDATE ทับค่าใหม่ (มีผลใน 5 วิ)

---

## 7. ลำดับที่แนะนำ

1. **Deploy ก่อน** — `mirror_edge_zones` ยังว่าง = ฟีเจอร์ปิด ปลอดภัย ไม่เปลี่ยนพฤติกรรมเดิม
2. ดู log `ZeroSig` 1–2 วัน → ยืนยันว่าเลนไหน/ฝั่งไหนเป็นขอบ (ข้อ 2 วิธี B)
3. `UPDATE` ค่าเข้าไป → ตรวจ log ว่าขึ้น `[EdgeMirror] mirror...` (ข้อ 5)

---

## หมายเหตุสำคัญ

- น้ำหนักที่ได้จาก mirror เป็น **ค่าประมาณ** (ติด flag `is_estimated`) — ใช้นับรถ/สถิติ/ESAL เท่านั้น
  **ห้ามใช้เป็นหลักฐานน้ำหนักเกิน** (record พวกนี้ถูกบังคับ `is_overweight = 0`)
- ดูหลักการทำงานเต็มได้ที่ [straddling-detection.md](straddling-detection.md)
