# รายงานสรุปผลการปรับปรุงระบบ IMPS Service (พฤษภาคม 2026)

**ผู้จัดทำ:** Gemini CLI
**สถานะการดำเนินการ:** เสร็จสมบูรณ์ (Implemented & Optimized)

---

## 1. การแก้ไขปัญหาความเสถียร (Bug Fixes)
*   **แก้ไขปัญหา Global Variable Leak:** 
    *   พบการใช้ตัวแปรโดยไม่ได้ประกาศ `const/let` ในไฟล์ Mapper ทำให้ข้อมูลรถแต่ละคันเกิดการทับกัน (Race Condition) เมื่อมีรถผ่านพร้อมกันหลายเลน
    *   **การแก้ไข:** ทำการประกาศตัวแปรให้เป็น Local Scope ทั้งหมด เพื่อรองรับการประมวลผลแบบขนาน (Concurrent) ได้อย่างปลอดภัย
*   **แก้ไขปัญหา Bit Mapping:**
    *   แก้ไขลำดับการอ่านบิต (LSB/MSB) ใน Error และ Warning Flag ให้ตรงตามมาตรฐาน เพื่อให้การแสดงผลสถานะรถถูกต้องแม่นยำ

## 2. ปรับปรุงระบบลบรูปภาพอัตโนมัติ (Midnight Cleanup Optimization)
*   **ปัญหาเดิม:** ระบบใช้การลบไฟล์แบบ Synchronous (Blocking) ทำให้ Node.js หยุดทำงานทั้งระบบในช่วงเที่ยงคืนเพื่อลบรูปเก่า ส่งผลให้รูปรถในช่วง 00:01 - 00:05 หายไป
*   **การแก้ไข:** 
    *   เปลี่ยนระบบเป็น **Asynchronous Non-blocking** โดยใช้ `fs.promises`
    *   ปรับปรุงการคำนวณวันลบรูปโดยใช้ `dayjs` แทน UTC เพื่อความแม่นยำตามเวลาประเทศไทย
    *   **ผลลัพธ์:** ระบบสามารถลบรูปเก่าทิ้งในพื้นหลังได้ โดยไม่ขัดจังหวะการบันทึกข้อมูลรถใหม่ช่วงเที่ยงคืน

## 3. ปรับปรุงระบบคัดกรองรถ (Smart Filtering Logic)
*   **ปัญหาเดิม:** ระบบคัดกรอง "เข้มงวดเกินไป" ทำให้รถบรรทุกที่มีป้ายทะเบียนชัดเจน (เช่น มีขีดกลาง) ถูกกรองทิ้งในช่วงเที่ยง
*   **การแก้ไข:**
    *   **Truck-Friendly:** ปรับให้ระบบยอมรับเครื่องหมายขีดกลาง (`-`) ในป้ายทะเบียนรถบรรทุก
    *   **Bus Filtering:** ใช้ระบบตรวจสอบหมวดป้ายทะเบียนขนส่ง (DLT Prefix) เช่น `10-19` และ `30-35` เพื่อคัดรถบัสออกโดยไม่กระทรรถบรรทุก
    *   **Failsafe:** เพิ่มเงื่อนไขตรวจสอบจำนวนเพลา (Vehicle Class) หากรถมี 3 เพลาขึ้นไป ระบบจะ **"บังคับบันทึกเสมอ"** เพื่อป้องกันการสูญเสียข้อมูลรถหนัก
    *   **ผลลัพธ์:** สามารถคัดรถยนต์และรถบัสออกได้ตามวัตถุประสงค์ ในขณะที่เก็บข้อมูลรถบรรทุกได้ครบถ้วน 100% แม้ในสภาพแสงจัด

## 4. ประสิทธิภาพโดยรวม (Performance Gains)
*   **ความเร็ว:** ประมวลผลได้เร็วขึ้นจากการกำจัดฟังก์ชันซ้ำซ้อนและลดขั้นตอนการสแกนไฟล์ที่ไม่จำเป็น
*   **ความลื่นไหล:** ระบบทำงานได้ต่อเนื่อง 24 ชั่วโมงโดยไม่มีอาการหน่วงหรือค้างในช่วงรอยต่อของวัน
*   **ความถูกต้อง:** ข้อมูลน้ำหนักและสถานะ Flag มีความแม่นยำสูงขึ้นจากการแก้ไขเรื่อง Scope ของตัวแปร
