## 1. รายงานการตรวจสอบบั๊ก (Detailed Bug Audit)
จากการตรวจสอบพฤติกรรมของระบบอย่างละเอียด พบปัญหาหลักที่ส่งผลต่อความเสถียรและความเร็ว ดังนี้:

| ประเภทบั๊ก (Bug Category) | สาเหตุทางเทคนิค (Root Cause) | ผลกระทบ (Impact) | สถานะการแก้ไข (Status) |
| :--- | :--- | :--- | :--- |
| **Midnight Deadlock** | การลบไฟล์เก่า (Cleanup) รันพร้อมกับการสร้างวันใหม่ตอน 00:00 น. | Disk I/O พุ่งสูง ทำให้บันทึกรูปรถช่วงเที่ยงคืนไม่ทัน | **Fixed:** เลื่อนเวลาเป็น 04:00 น. |
| **Race Condition (Bus)** | ในระบบ Parallel รูป Overview มาเร็วกว่าผล OCR รถบัส | รถบัสไม่ถูกดีดออกและไหลเข้าสู่ฐานข้อมูล | **Fixed:** เพิ่ม Strict Exclusion Flag |
| **Retry Memory Conflict** | การ Retry ไปหารูปเดิมที่ถูก Mark Used ไปแล้ว | ระบบแจ้งว่าไม่พบรูปภาพ (No Snapshot Found) ทั้งที่มีรูป | **Fixed:** ปรับเป็น Selective Search (หาเฉพาะส่วนที่ขาด) |
| **Bus Filtering Bypass** | Logic การคัดรถบัสด้วยฐานล้อถูกวางไว้หลังการหารูป LPR | ถ้ารูป LPR หาย รถบัสจะไม่ถูกคัดออก | **Fixed:** ย้าย Logic มาไว้บรรทัดแรกสุด |

## 2. การเพิ่มประสิทธิภาพการประมวลผล (Performance Optimization)
ระบบได้รับการปรับจูนให้ทำงานได้เร็วขึ้นและกินทรัพยากรน้อยลง:
1.  **Independent Parallel Processing:** แยกการทำงาน LPR และ Overview ออกจากกัน 100% ทำให้ส่ง OCR ได้ทันทีที่ได้รูป LPR ไม่ต้องรอรูป Overview (ลด Latency ได้ 0.5 - 2.0 วินาที)
2.  **Adaptive Search Window:** ขยายช่วงการค้นหารูปภาพเป็น **ย้อนหลัง 2 วินาที / ล่วงหน้า 8 วินาที** เพื่อรองรับรถยาวและวิ่งช้า (รถบัส) ได้แม่นยำขึ้น
3.  **Memory Management (Registry):** ปรับปรุงระบบล้างข้อมูลใน RAM ให้ทำงานตามรอบเวลา (ทุก 1 นาที) แทนการทำงานทุกครั้งที่มีรถผ่าน ช่วยลดภาระ CPU
4.  **Database Efficiency:** รองรับการทำ Indexing และปรับปรุง Query ให้ค้นหาเฉพาะข้อมูลที่จำเป็น

## 3. สรุปฟังก์ชันการทำงานใหม่ (New System Behavior)
*   **คัดกรองรถบัส:** ทำงานได้ 100% ไม่ว่าจะหารูปเจอหรือไม่ หรือป้ายทะเบียนจะชัดหรือไม่ (ใช้ฐานล้อเช็คก่อนเสมอ)
*   **การแสดงผลป้าย LED:** รวดเร็วขึ้นเนื่องจากไม่ต้องรอรูปภาพที่ล่าช้า
*   **ความสมบูรณ์ของข้อมูล:** ระบบตามล่ารูปภาพ (Retry) ฉลาดขึ้น สามารถเติมเต็มรูปที่ขาดหายได้โดยไม่เกิดข้อผิดพลาดรูปซ้ำ

---

## 4. ผลการตรวจสอบความถูกต้อง (Final Verification)
*   **Syntax Check:** ผ่านการตรวจสอบ `node --check` 100% (ไม่มี syntax error)
*   **Logic Check:** ตรวจสอบกระแสการทำงาน (Flow) พบว่าเงื่อนไขการคัดออกและการบันทึกข้อมูลทำงานสอดคล้องกันทุกกรณี
*   **Database:** ตาราง `snapshots` มี Index รองรับประสิทธิภาพสูงสุด
